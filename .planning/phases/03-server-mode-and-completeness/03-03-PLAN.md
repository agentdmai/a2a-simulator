<objective>
Close three integration wiring gaps identified in the v1.0 Milestone Audit that prevent server-mode features from working at runtime.

Purpose: The server broadcasts `incoming-task`, `task-canceled`, and `reply-sent` SSE events, but the browser never receives them because `useSSE.ts` has no `addEventListener` calls for these event names. This makes the entire server-mode UI (SRVR-02/03/04/05) dead at runtime despite all components being correctly implemented. Additionally, the SuccessBanner after Agent Card save never displays due to a mount/unmount race.

Output: Three files patched with targeted wiring fixes -- no structural changes.
</objective>

    **In src/client/components/AgentCardEditorDrawer.tsx:**

    1. Add `onSaveSuccess?: (message: string) => void` to `AgentCardEditorDrawerProps`.
    2. In `handleSave`, replace `setSuccessMessage('Agent Card updated')` with `onSaveSuccess?.('Agent Card updated')`.
    3. Remove the local `successMessage` state (line 24).
    4. Remove the `SuccessBanner` render from the drawer JSX (line 66).
    5. Remove the `SuccessBanner` import (line 7).
    6. Keep `onClose()` call after the save — the drawer should still close.

    **In src/client/App.tsx:**

    1. Add back `successMessage` state: `const [successMessage, setSuccessMessage] = useState<string | null>(null);`
    2. Add a `handleSaveSuccess` callback: `const handleSaveSuccess = useCallback((msg: string) => { setSuccessMessage(msg); }, []);`
    3. Pass it to AgentCardEditorDrawer: `<AgentCardEditorDrawer open={editorOpen} onClose={() => setEditorOpen(false)} onSaveSuccess={handleSaveSuccess} />`
    4. Render SuccessBanner in the main content area (NOT inside the drawer): `{successMessage && <SuccessBanner message={successMessage} onDismiss={() => setSuccessMessage(null)} />}` — place it above `<ChatPanel />` in the right panel div.
