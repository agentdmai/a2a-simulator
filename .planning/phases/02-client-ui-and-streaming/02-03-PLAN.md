The logic flow:
- First `status-update` with state=working and a message: find existing streaming message for this task (one with `isStreaming: true`), if exists update its text, otherwise add new message with `isStreaming: true`.

Do NOT add new action types to the ConnectionAction union for this -- handle it entirely within the existing TASK_EVENT case.

4. In `src/client/context/ConnectionContext.tsx`, in the `TASK_EVENT` case:

     if (action.payload.rawRequest || action.payload.rawResponse) {
